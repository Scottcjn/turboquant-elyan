# Contributing to TurboQuant Elyan

Thanks for your interest in contributing! 

## How to Contribute

### Bug Reports
1. Search existing issues to avoid duplicates
2. Open a new issue with clear reproduction steps
3. Include Python version, OS, and relevant config

### Code Contributions
1. **Fork** this repository
2. **Create a branch**: `git checkout -b feature/your-feature`
3. **Write code** with type hints and docstrings
4. **Add tests** for new functionality
5. **Submit a PR** with a clear description

### Code Style
- Python 3.9+ compatible
- Use type hints for function signatures
- Follow PEP 8 naming conventions
- Add docstrings to all public functions

### Testing
- Run `pytest tests/` before submitting
- New features require new tests
- All tests must pass on CI

### Quality Checklist
- [ ] Code follows project style
- [ ] New code has tests
- [ ] All existing tests pass
- [ ] No hardcoded secrets or API keys
- [ ] Documentation updated if needed

## Code of Conduct
Be respectful. Constructive feedback only. No harassment.
